# Implementation Plan: Plugin Distribution & Easy Installation

**Branch**: `001-copilot-opencode-adapter` | **Date**: 2026-05-03 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `specs/002-plugin-distribution/spec.md`

## Summary

Package and distribute the OpenCopilot plugin so users can install it with `npx opencopilot install`. The core technical approach is:

1. **Bundle**: A `bun build` script inlines all `src/` code into a single self-contained `dist/opencopilot.ts` file, keeping `@opencode-ai/plugin` and `js-yaml` as externals (provided by OpenCode's own `bun install`).
2. **Installer**: A `bin/install.js` Node.js CLI script fetches `dist/opencopilot.ts` from GitHub Releases and writes it to `.opencode/plugins/opencopilot.ts`.
3. **Distribution**: GitHub Actions releases on tag push (`v*.*.*`), creates a GitHub Release with the bundled file as an asset, and publishes to npm.
4. **README**: Updated to show `npx opencopilot install` as the primary install instruction.

---

## Technical Context

**Language/Version**: TypeScript 5.8 (source); JavaScript ES2022 (installer script)  
**Primary Dependencies**: Bun (bundler + test runner), Node.js 18+ (installer runtime via `npx`), `@opencode-ai/plugin` (external), `js-yaml` (external)  
**Storage**: File system — writes `.opencode/plugins/opencopilot.ts` and optionally `.opencode/package.json`  
**Testing**: `bun test` (existing test suite) + new integration tests for installer and bundle  
**Target Platform**: macOS, Linux, Windows with WSL (Node.js 18+ required for installer)  
**Project Type**: CLI tool (installer) + plugin artifact distribution  
**Performance Goals**: Install completes in under 10 seconds under normal network conditions (FR-008)  
**Constraints**: Zero external npm dependencies for the installer; no build step required at install time (FR-004)  
**Scale/Scope**: Single-file distribution; one binary (`npx opencopilot install`); one release asset per tag

## Architecture Decisions (Summary)

See `research.md` for full rationale. Summary:

| Decision | Choice |
|---|---|
| **Installer source** | GitHub Releases latest asset URL; falls back to embedded `dist/opencopilot.ts` in npm package if available |
| **Installer runtime** | Node.js 18+ via `#!/usr/bin/env node` shebang; works with `npx` universally |
| **Bundle tool** | `bun build` with `--external @opencode-ai/plugin --external js-yaml` |
| **Output format** | Single `.ts` file (Bun loads it natively; `.ts` extension required for OpenCode plugin path) |
| **Release trigger** | Git tag push `v*.*.*` |
| **Plugin file path** | Exactly `.opencode/plugins/opencopilot.ts` (FR-001, non-negotiable) |
| **External deps handling** | Kept external; installer writes/updates `.opencode/package.json` so OpenCode `bun install` provides them |
| **JSR** | Deferred to v2 (spec Assumptions) |

## Testing Strategy

### Unit Tests (`tests/installer/bundle.test.ts`)

| Test | Description |
|---|---|
| Bundle file exists | `dist/opencopilot.ts` is created by `bun run bundle` |
| Version banner | First line matches `// opencopilot v{semver}` |
| No relative imports | File does not contain `../../src/` |
| Default export | File exports `OpenCopilotPlugin` as default |
| External imports only | Only allowed imports are `@opencode-ai/plugin`, `js-yaml`, `node:*` built-ins |

### Integration Tests (`tests/installer/install.test.ts`)

| Test | Description |
|---|---|
| Fresh install | Creates `.opencode/plugins/` dir and writes `opencopilot.ts` |
| File already exists, no --force | Prompts user; does not overwrite when user says 'n' |
| File already exists, --force | Overwrites without prompting |
| Symlink at target path | Exits 1 with symlink warning; does not write |
| Missing parent dirs | Creates intermediate directories automatically |
| `.opencode/package.json` creation | Creates file with required deps when it doesn't exist |
| `.opencode/package.json` merge | Adds missing deps without removing existing entries |
| Network error | Exits 1 with error message + fallback URL |
| Success message | Outputs version and file path on success |

## Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Bun bundler output is `.js` not `.ts` | Low | High | Use `--outfile dist/opencopilot.ts`; Bun respects the extension on output files |

| GitHub Releases `/releases/latest/download/` redirect not working | Low | Medium | Test redirect chain in T008; document versioned URL as fallback |
